Digital Image Basics

What is a Digital Image?

A digital image is composed of pixels with intensity values derived from a 2-D function f(x, y), where x and y represent spatial coordinates, and f denotes intensity or gray level.

Key Concepts

Pixel

The smallest addressable unit in an image. Each pixel contains intensity information that determines the brightness or color at that specific location.

RGB Color Model

A three-tuple (r, g, b) representing three color channels - Red, Green, and Blue. This is the most common color representation in digital systems.

Bit Depth

Defines the range of values (2^n) available for each pixel. For example, 8-bit depth provides 0–255 possible values per color channel.

Number of possible values = 2^n
where n = number of bits per pixel

Image Storage

  • Grayscale Images: Stored as a 2-D array (width × height)
  • RGB Images: Stored as a 3-D array (width × height × 3)
  • Pixel Values: Can be represented as integers (0–255) or floats (0–1)
  • Common Formats: JPEG, PNG, GIF - each containing metadata

Digitization Process

Sampling: Discretizes spatial coordinates into a grid of pixels

Quantization: Discretizes intensity values into discrete levels

Image Processing Pipeline

  1. Acquisition/Storage: Captures and stores images (e.g., JPEG format)
  2. Pre-processing: Grayscale conversion, enhancement, restoration
  3. Segmentation: Extract objects (challenges: noise, occlusion)
  4. Morphological Processing: Correct structural defects
  5. Information Extraction: Convert unstructured to structured data
  6. Interpretation: Classification and object detection
  7. Compression: Reduce size (may affect quality)
Image Formation Model

An image can be represented as:

f(x, y) = i(x, y) · r(x, y)

Where:

  • i(x, y): Illumination component
  • r(x, y): Reflectance component
  • Gray Level: l = f(x, y), ranging from [0, L–1]

Image Sampling and Quantization

Overview

Converting continuous images f(x, y) into digital format involves two critical steps: sampling (spatial discretization) and quantization (amplitude discretization).

Sampling

Definition

Discretizes spatial coordinates to form a grid of pixels. The sampling rate controls the number of pixels, directly impacting spatial resolution.

Higher sampling rate = More pixels = Better resolution = Increased storage

Types of Sampling

Down-sampling

Reduces pixel count by selecting every M-th value: y(n) = x(Mn)

Example

Input: x(n) = {1, -1, 2, 4, 0, 3, 2, 1, 5} with M=2

Output: y(n) = {1, 2, 0, 2, 5}

Up-sampling

Increases pixel count by inserting L-1 zeros between values

Example

Input: x(n) = {1, 2, 4, -2, 3, 2, 1} with L=2

Output: y(n) = {1, 0, 2, 0, 4, 0, -2, 0, 3, 0, 2, 0, 1}

Up-sampling Techniques

  • Nearest Neighbor: Duplicates nearby pixel values
  • Bilinear Interpolation: Uses four neighboring pixels for smoother results
  • Cubic Interpolation: Provides highest accuracy for improved image quality

Quantization

Definition

Converts continuous intensity values into discrete levels (e.g., 0–255 for 8-bit images).

L = 2^k
where k = bits per pixel (BPP)

Examples:

  • k=2 → L=4 levels
  • k=8 → L=256 levels (standard grayscale)
Quantization Example

Sequence: 550, 600, -100, 150, -300, 900, 0, 850 with 2 bits:

  • Levels: L = 2² = 4
  • Range: 900 - (-300) = 1200
  • Quantization step (δ): 1200 ÷ 4 = 300
  • Level ranges: [-300 to 0], [1 to 300], [301 to 600], [601 to 900]

Impact on Image Quality

  • Spatial Resolution: Higher sampling rate = sharper image
  • Intensity Resolution: More gray levels = richer details
  • Storage: M×N image with k bits = M×N×k bits total
  • False Contours: Occur with low quantization levels

Point Operations

Overview

Point operations modify pixel values without altering image size, geometry, or local structure. The new pixel value depends only on its previous value, mapped by function f(a).

Enhancement Techniques

Negative Transformation

Produces the negative of an image using the formula:

s = (L-1) - r
For 8-bit: s = 255 - r
Example

If r = 0 (black) → s = 255 (white)

If r = 255 (white) → s = 0 (black)

Pixel value 100 → 255 - 100 = 155

Applications: Enhances white/gray details in dark regions, increases contrast

Gray-scale Threshold Transform

Converts a gray-scale image to a binary image using a threshold T.

s = { L-1 if r ≥ T; 0 if r < T }

Applications: Object detection, image segmentation

Example

For T = 128 and L = 256:

r = 150 → s = 255

r = 100 → s = 0

Clipping Transform

Highlights pixels within a specified range [min, max].

s = { L-1 if min ≤ r ≤ max; 0 otherwise }

Purpose: Focus on specific intensity ranges, remove background

Example

For min = 50, max = 200:

r = 75 → s = 255

r = 25 → s = 0

Bit-plane Slicing

Analyzes individual bits in pixel values by separating each bit into a binary image.

Applications: Image enhancement, compression

Example

For pixel value 153 (binary: 10011001):

Bit-plane 7: 1, Bit-plane 6: 0, Bit-plane 5: 0, etc.

Color Image Processing

Overview

Color is a powerful descriptor in automated image analysis, simplifying object identification and extraction. Human vision can distinguish thousands of color shades but only 20-30 gray shades, making color highly effective for image processing.

Pseudo-Color Processing

Definition

Converts a gray-scale image into a color image by assigning colors to different intensity levels.

Purpose: Enhances visual interpretation of gray-scale images lacking detail or contrast.

Applications: Medical imaging, satellite imagery, thermal imaging.

Techniques: Intensity slicing, color mapping using lookup tables (LUT).

Example

A thermal image displayed in pseudo-color to highlight temperature variations.

Full-Color Processing

Definition

Processes images already in color (e.g., RGB), manipulating color components directly.

Purpose: Enhances or analyzes color information while preserving natural appearance.

Applications: Digital photography, video processing, computer vision.

Techniques: Color space transformations, color balancing, histogram equalization, edge detection in color images.

Color Fundamentals

  • Color Spectrum: White light splits into a continuous spectrum: violet, indigo, blue, green, yellow, orange, red.
  • Color Perception: Depends on light reflected by an object.
  • Achromatic Light: Light without color (grayscale), characterized by intensity.
  • Chromatic Light Quantities: Radiance, luminance, brightness.
  • Primary Colors: Red (700 nm), Green (546.1 nm), Blue (435.8 nm).
  • Secondary Colors: Magenta (R+B), Cyan (G+B), Yellow (R+G).
  • Color Characteristics: Hue, saturation, brightness, chromaticity.

Color Models

Definition

A color model specifies colors in a standardized 3-D coordinate system, where each color is a single point.

Common Models:

  • RGB: Used in color monitors and video cameras.
  • CMYK: Used in printing devices.
  • HSI: Aligns with human color perception, separates color and grayscale information.
RGB Color Model

Based on a Cartesian coordinate system with R, G, B values normalized to [0,1]. The color cube has primary colors at corners, secondary colors at other corners, black at (0,0,0), white at (1,1,1), and grayscale along the diagonal.

Image Segmentation

Overview

Image segmentation partitions an image into multiple regions to locate objects and boundaries, simplifying object identification and analysis.

Fundamentals

Let R denote the entire image region. Segmentation divides R into n sub-regions (R1, R2, ..., Rn) with properties like completeness, disjointness, homogeneity within regions, and heterogeneity between adjacent regions.

Segmentation Properties

  • Discontinuity: Partitions based on abrupt intensity changes (e.g., edges).
  • Similarity: Partitions based on similar intensity or texture according to predefined criteria.

Discontinuity-Based Segmentation

Methods

  • Isolated Point Detection: Uses second-order derivatives (Laplacian) to detect points where the filter response exceeds a threshold.
  • Line Detection: Uses second-order derivatives for thinner lines. Masks detect lines in specific directions.
  • Edge Detection: Identifies edges using first-order (e.g., Sobel) or second-order (e.g., Laplacian) derivatives.

Similarity-Based Segmentation

Methods

  • Thresholding: Segments based on intensity ranges. Includes global, variable, and adaptive thresholding.
  • Region Growing: Starts from seed pixels and grows regions by adding similar pixels.
  • Region Splitting and Merging: Divides heterogeneous regions and merges similar adjacent regions.
Global Thresholding Procedure
  1. Select initial threshold T (e.g., average intensity)
  2. Segment into G1 (intensity > T) and G2 (intensity ≤ T)
  3. Compute mean intensities m1 and m2 for G1 and G2
  4. Update T as (m1 + m2)/2
  5. Repeat until T stabilizes

Region-Based Segmentation

Overview

Region-based segmentation directly identifies regions using connectivity of similar pixels, offering robustness to noise compared to edge or threshold methods.

Region Growing

Definition

Grows regions from seed points by adding connected pixels meeting a predicate Q (e.g., intensity similarity).

Procedure:

  1. Identify connected components in seed image S
  2. Form fQ image: fQ(x,y)=1 if input satisfies Q, else 0
  3. Append 8-connected 1-valued pixels in fQ to seeds in S
  4. Label connected components in resulting image g

Region Splitting and Merging

Definition

Divides inhomogeneous regions and merges homogeneous ones.

Procedure:

  • Split: If region R is inhomogeneous (p(R)=false), divide into four sub-regions
  • Merge: If adjacent regions Ri and Rj are homogeneous (p(Ri ∪ Rj)=true), merge them
  • Stop when no further splitting or merging is possible

Morphological Operations

Definition

Processes binary images based on shape using structuring elements to refine segmentation results.

Basic Operations:

  • Erosion (f ⊖ s): Shrinks foreground, splits joined objects
  • Dilation (f ⊕ s): Expands foreground, fills holes

Spatial Filtering

Fundamentals

Spatial filtering modifies an image by replacing each pixel value with a function of its value and its neighbors. Filters can be linear (e.g., convolution) or nonlinear (e.g., median filter).

Convolution and Correlation

Convolution

Slides a kernel over the image, computing a weighted sum. For 1D: y[n] = Σ h[m] · x[n-m]

Example

Image I = {0,0,1,0,0}, kernel k = {3,2,8}

After zero-padding and kernel rotation, output y = {0,0,3,2,0}

Correlation

Similar to convolution but without kernel rotation. Used in template matching.

Non-Linear Filters

Order-Statistics Filters

Based on ordering pixel values in a neighborhood.

  • Median Filter: Replaces pixel with median value, effective for salt-and-pepper noise
  • Min Filter: Replaces with minimum value, good for salt noise
  • Max Filter: Replaces with maximum value, good for pepper noise

Image Restoration

Definition

Image restoration reconstructs original images from degraded versions by identifying and reversing degradation processes like noise or blur.

Noise in Images

Sources: Image acquisition (e.g., sensor noise), transmission interference

Model: g(x,y) = f(x,y) + η(x,y), where η is noise

Noise Removal Techniques

Spatial Domain Filters

  • Mean Filter: Averages neighborhood pixels
  • Median Filter: Replaces with median value, effective for impulse noise
  • Adaptive Filters: Adjust based on local statistics

Frequency Domain Filters

Used for periodic noise removal, e.g., band-reject filters to eliminate specific frequencies.
